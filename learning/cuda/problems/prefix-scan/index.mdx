## Background

**Prefix scan** (or scan, or parallel prefix) is a fundamental parallel algorithm that computes running prefixes of a sequence. Given an input array `[a, b, c, d, e, ...]` and an associative binary operator (like addition), scan produces:

- **Exclusive scan**: `[0, a, a+b, a+b+c, a+b+c+d, ...]` — each output is the sum of all preceding elements
- **Inclusive scan**: `[a, a+b, a+b+c, a+b+c+d, ...]` — each output includes the element itself

The exclusive scan is more useful for computing output indices in stream compaction and scatter operations.

The **Blelloch scan algorithm** achieves O(N) work (same as sequential) with O(log N) parallel steps. It uses two phases:
1. **Upsweep** (reduce): Build a binary tree of partial sums, bottom-up
2. **Downsweep** (distribute): Propagate prefix sums top-down, computing the exclusive prefixes

## What to Implement

Implement an exclusive prefix sum over an array of `n` unsigned integers. Your solution should:

1. Implement the Blelloch upsweep phase: for each level, add sibling pairs
2. Set the final element to 0 (exclusive scan base case)
3. Implement the downsweep phase: propagate sums, swapping children appropriately
4. Use warp shuffle instructions for the final warp reduction
5. Handle arrays larger than a single block using a three-pass approach

## Constraints

- `n` is a power of 2, `1 ≤ n ≤ 2^20`
- Block size must be 256 threads
- Use shared memory for intra-block communication
- The result must match a sequential CPU scan exactly

#include <cuda_runtime.h>

#define BLOCK_SIZE 256

// Single-block exclusive prefix scan using Blelloch algorithm
__global__ void blockScan(const unsigned int* in, unsigned int* out, int n) {
    __shared__ unsigned int tree[2 * BLOCK_SIZE];

    int tid = threadIdx.x;
    int i = blockIdx.x * BLOCK_SIZE + tid;

    // Step 1: Load input into shared memory
    tree[BLOCK_SIZE + tid] = (i < n) ? in[i] : 0;
    __syncthreads();

    // Step 2: Upsweep — build partial sums bottom-up
    for (int d = 0; d < 8; d++) {  // log2(256) = 8 levels
        int stride = 1 << (d + 1);  // 2, 4, 8, 16, 32, 64, 128, 256
        if ((tid & (stride - 1)) == (stride / 2)) {
            tree[tid] += tree[tid - stride / 2];
        }
        __syncthreads();
    }

    // Step 3: Set last element to 0 (exclusive scan base case)
    if (tid == 0) tree[2 * BLOCK_SIZE - 1] = 0;
    __syncthreads();

    // Step 4: Downsweep — propagate sums top-down
    for (int d = 7; d >= 0; d--) {  // Reverse order
        int stride = 1 << (d + 1);
        if ((tid & (stride - 1)) == (stride / 2)) {
            unsigned int t = tree[tid - stride / 2];
            tree[tid - stride / 2] = tree[tid];
            tree[tid] += t;
        }
        __syncthreads();
    }

    // Step 5: Store exclusive prefix to output
    out[i] = tree[BLOCK_SIZE + tid];
}

void scanPass1(const unsigned int* d_in, unsigned int* d_blocks,
               unsigned int* d_sums, int n) {
    int numBlocks = (n + BLOCK_SIZE - 1) / BLOCK_SIZE;
    blockScan<<<numBlocks, BLOCK_SIZE>>>(d_in, d_blocks, n);

    // Extract block sums (last element of each block)
    __global__ void extractSums(const unsigned int* blocks,
                                unsigned int* sums, int numBlocks) {
        int bid = blockIdx.x * blockDim.x + threadIdx.x;
        if (bid < numBlocks) {
            sums[bid] = blocks[(bid + 1) * BLOCK_SIZE - 1];
        }
    }
    extractSums<<<(numBlocks + 255) / 256, 256>>>(d_blocks, d_sums, numBlocks);
}

               unsigned int* d_out, int n) {
    int numBlocks = (n + BLOCK_SIZE - 1) / BLOCK_SIZE;

    __global__ void addOffsets(unsigned int* blocks, const unsigned int* offsets,
                               int n, int numBlocks) {
        int tid = blockIdx.x * blockDim.x + threadIdx.x;
        int bid = tid / BLOCK_SIZE;
        int local_idx = tid % BLOCK_SIZE;

            blocks[tid] += offsets[bid - 1];
        }
    }
    addOffsets<<<(n + 255) / 256, 256>>>(d_blocks, d_offsets, n, numBlocks);

    // Copy result back
    __global__ void copyOut(const unsigned int* blocks, unsigned int* out, int n) {
        int tid = blockIdx.x * blockDim.x + threadIdx.x;
        if (tid < n) out[tid] = blocks[tid];
    }
    copyOut<<<(n + 255) / 256, 256>>>(d_blocks, d_out, n);
}

## Approach

The Blelloch scan algorithm consists of two phases:

**Upsweep (reduce):**
```cpp
for (int d = 0; d < log2(BLOCK_SIZE); d++) {
    int stride = 1 << (d + 1);
    if ((tid & (stride - 1)) == (stride / 2)) {
        tree[tid] += tree[tid - stride / 2];
    }
}
```
This builds a binary tree from bottom to top. At each level, the right child of each pair adds the left child's value.

**Downsweep (distribute):**
```cpp
tree[2 * BLOCK_SIZE - 1] = 0;  // Base case for exclusive scan
for (int d = log2(BLOCK_SIZE) - 1; d >= 0; d--) {
    int stride = 1 << (d + 1);
    if ((tid & (stride - 1)) == (stride / 2)) {
        unsigned int t = tree[tid - stride / 2];
        tree[tid - stride / 2] = tree[tid];
        tree[tid] += t;
    }
}
```
This propagates prefix sums from top to bottom. The right child gets the parent's sum; the left child keeps its sum.

**Three-pass algorithm for large arrays:**
1. Scan each block independently
2. Scan the block sums (one block)
3. Add the offset from each block's sum to all elements in that block

## Complexity

- **Time**: O(N) work, O(log N) steps for single block
- **Shared memory**: 2 × BLOCK_SIZE integers = 2048 bytes for BLOCK_SIZE=256
- **Three-pass overhead**: 3 kernel launches, extra global memory traffic

## Key Insights

1. **Blelloch vs Hillis-Steele**: Blelloch is work-efficient (O(N) work) while Hillis-Steele is not (O(N log N) work). Blelloch has more complex dependencies but better asymptotic performance.

2. **Exclusive vs Inclusive**: Exclusive scan outputs 0 at position 0, while inclusive scan outputs the first element. Exclusive is useful for computing output indices.

3. **Warp optimization**: For the final downsweep step, use warp shuffle instead of shared memory:
   ```cpp
   for (int offset = 16; offset > 0; offset >>= 1)
       val += __shfl_up_sync(0xffffffff, val, offset);
   ```

## Common Mistakes

- **Wrong tree indexing**: The tree array needs 2 × BLOCK_SIZE elements. Leaves are at indices [BLOCK_SIZE, 2*BLOCK_SIZE).
- **Missing syncthreads**: Each level of upsweep and downsweep requires synchronization before the next level.
- **Off-by-one in upsweep**: The loop runs for log2(BLOCK_SIZE) iterations, not log2(BLOCK_SIZE)-1.
- **Not handling small N**: When n < BLOCK_SIZE, pad with zeros to avoid reading out of bounds.
