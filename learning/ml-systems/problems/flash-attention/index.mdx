## Background

Standard scaled dot-product attention materialises the full N×N attention matrix in HBM (GPU global memory). For a sequence of length N and head dimension D, this costs O(N²) memory and O(N²) HBM reads/writes — the dominant bottleneck for long contexts.

**Flash Attention** (Dao et al., 2022) eliminates the N×N materialisation entirely by computing attention in **tiles** that fit in SRAM. The key insight is the **online softmax** trick: you can compute softmax incrementally, updating a running maximum `m` and a running sum `l`, without ever storing the full row. The output `O` is updated tile-by-tile:

```
For each tile K_j, V_j:
  S_j   = Q * K_j^T / sqrt(D)             [block scores]
  m_new = max(m_prev, rowmax(S_j))
  P_j   = exp(S_j - m_new)                [unnormalised]
  l_new = exp(m_prev - m_new) * l_prev + rowsum(P_j)
  O     = (exp(m_prev - m_new) * O_prev + P_j * V_j) / l_new
```

This keeps memory at O(N) and cuts HBM traffic by ~5–20× for long sequences.

## Constraints

- `seq_len` is a multiple of `block_size`
- `1 ≤ seq_len ≤ 2048`, `1 ≤ head_dim ≤ 128`, `1 ≤ block_size ≤ seq_len`
- Output must match standard attention (full materialisation) within `1e-4` absolute error
- No external libraries (use `<cmath>` only)

void flash_attention_forward(
    const float* Q, const float* K, const float* V,
    float* O, int seq_len, int head_dim, int block_size)
{
    const float scale = 1.0f / std::sqrt((float)head_dim);
    int num_kv_blocks = seq_len / block_size;

    std::vector<float> s(block_size), o_row(head_dim);

    for (int i = 0; i < seq_len; ++i) {
        const float* q = Q + i * head_dim;
        float m = -std::numeric_limits<float>::infinity();
        float l = 0.0f;
        std::fill(o_row.begin(), o_row.end(), 0.0f);

        for (int jb = 0; jb < num_kv_blocks; ++jb) {
            int j0 = jb * block_size;
            // Compute scores S = Q_i * K[j0..j0+block_size]^T * scale
            for (int jj = 0; jj < block_size; ++jj) {
                float dot = 0.0f;
                const float* k = K + (j0 + jj) * head_dim;
                for (int d = 0; d < head_dim; ++d) dot += q[d] * k[d];
                s[jj] = dot * scale;
            }
            // Online softmax update
            float m_new = m;
            for (int jj = 0; jj < block_size; ++jj)
                m_new = std::max(m_new, s[jj]);

            float l_new = std::exp(m - m_new) * l;
            for (int jj = 0; jj < block_size; ++jj)
                l_new += std::exp(s[jj] - m_new);

            float correction = std::exp(m - m_new);
            for (int d = 0; d < head_dim; ++d) {
                float pv = 0.0f;
                for (int jj = 0; jj < block_size; ++jj)
                    pv += std::exp(s[jj] - m_new) * V[(j0 + jj) * head_dim + d];
                o_row[d] = correction * o_row[d] + pv;
            }
            m = m_new;
            l = l_new;
        }
        float* out = O + i * head_dim;
        for (int d = 0; d < head_dim; ++d)
            out[d] = o_row[d] / l;
    }
}

## Approach

Flash Attention avoids materialising the N×N attention matrix by using the **online softmax** recurrence. For each query row `Q_i`, iterate over KV tiles while maintaining running statistics `m` (max score so far) and `l` (normalisation factor):

```
For each KV tile j:
  Compute S = Q_i · K_j^T / sqrt(d)          [block scores]
  m_new = max(m, rowmax(S))
  l_new = exp(m - m_new) * l + rowsum(exp(S - m_new))
  O     = (exp(m - m_new) * O + exp(S - m_new) · V_j) / l_new  [rescale + accumulate]
  m = m_new; l = l_new
Final: O_i = O  (already normalised)
```

The `exp(m - m_new)` correction rescales the previously accumulated output when a new maximum is found, keeping numerical stability without a second pass over the data.

## Complexity

- **Time**: O(N² × D) — same as standard attention
- **Memory**: O(N × D) — no N×N matrix materialised, only the output and per-query running state
- **HBM reads**: O(N × D) vs O(N²) for standard — the key practical win

## Key Insight

The online softmax trick works because `softmax(x)_i = exp(x_i - max(x)) / sum(exp(x_j - max(x)))`. As we tile over the sequence, we can update the max and sum incrementally, correcting previous partial outputs by multiplying by `exp(m_old - m_new)` whenever the max increases.

## Common Mistakes

- **Dividing by l at each step instead of at the end**: Don't normalise intermediate O by l_new; only divide once at the end. If you divide at each step, the correction factor doesn't work correctly.
- **Initialising m = 0 instead of -inf**: With m = 0, the first tile's scores may all be negative, causing exp(S - m) to be very small and l underflows. Start with `m = -infinity`.
- **Not resetting m and l per query**: Each query row has independent running statistics. These must be initialised fresh for each `i`.
