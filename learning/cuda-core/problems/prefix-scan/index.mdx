## Background

**Prefix scan** (also called scan or parallel prefix) is one of the most fundamental parallel algorithms in GPU computing. Given an input array `[a, b, c, d, e, ...]` and an associative binary operator like addition, scan produces running prefixes:

- **Exclusive scan**: `[0, a, a+b, a+b+c, a+b+c+d, ...]` — each output is the sum of all preceding elements
- **Inclusive scan**: `[a, a+b, a+b+c, a+b+c+d, ...]` — each output includes the element itself

The exclusive scan variant is particularly valuable for computing output indices in stream compaction, scatter operations, and variable-length data indexing.

The **Blelloch scan algorithm**, named by Guy Blelloch, achieves O(N) total work (identical to the sequential algorithm) with O(log N) parallel steps. This dual optimality distinguishes it from the simpler Hillis-Steele algorithm, which achieves O(log N) steps but requires O(N log N) total work — a work-inefficient approach that wastes GPU compute throughput.

The Blelloch algorithm operates in two distinct phases:

1. **Upsweep (reduce phase)**: Build a binary tree of partial sums from bottom to top. At each level, sibling pairs combine, with the right child accumulating the left child's value. After log2(N) levels, the tree root holds the total sum.

2. **Downsweep (distribute phase)**: Propagate prefix sums from top to bottom. The root is initialized to 0 (for exclusive scan). At each level, the right child receives the parent's sum plus its sibling's sum; the left child receives the parent's sum unchanged. This distribution pattern ensures each leaf contains the exclusive prefix sum for its position.

**Work efficiency vs step efficiency** represents a critical design tradeoff in parallel algorithms. Work complexity counts the total operations across all processors; step complexity (also called depth or span) counts the number of parallel rounds. The ideal algorithm is both work-efficient (O(N) total work) and step-efficient (O(log N) steps). On GPUs with thousands of CUDA cores, work-inefficient algorithms waste arithmetic bandwidth even if they have few synchronization barriers.

The Hillis-Steele scan achieves O(log N) steps with N threads but performs N log N additions total. For N=1M, Hillis-Steele executes ~20M additions while Blelloch executes only ~2M additions — a 10× work difference that dominates runtime for large arrays despite Hillis-Steele's simpler dependencies.

## What to Implement

Implement an exclusive prefix sum over an array of unsigned integers using the Blelloch algorithm. Your solution must handle both single-block cases (N <= 256 elements) and large arrays requiring a three-pass grid-level scan.

**Single-block Blelloch scan:**

1. **Upsweep phase**: For each level d from 0 to log2(BLOCK_SIZE)-1:
   - Compute stride = 2^(d+1)
   - If thread index tid satisfies (tid % stride) == (stride/2), add the sibling at (tid - stride/2) to self
   - Synchronize before next level

2. **Base case**: Set the final element (rightmost leaf) to 0 for exclusive scan semantics

3. **Downsweep phase**: For each level d from log2(BLOCK_SIZE)-1 down to 0:
   - Compute stride = 2^(d+1)
   - If tid satisfies (tid % stride) == (stride/2), perform the swap-and-add operation:
     - Store the left child's value temporarily
     - Overwrite left child with right child's value
     - Add the temporary value to the right child
   - Synchronize before next level

4. **Warp optimization**: For the final downsweep step within each warp, use `__shfl_up_sync(0xffffffff, val, offset)` instead of shared memory to accumulate prefix sums — zero shared memory bank conflicts.

**Three-pass algorithm for N > BLOCK_SIZE:**

1. **Pass 1**: Each block scans its slice independently, storing results to output buffer and writing its total sum to a sums array
2. **Pass 2**: Recursively scan the sums array (one block, at most 256 elements)
3. **Pass 3**: Add the appropriate offset from the sums array to each element based on its block membership

## Constraints

- `n` is a power of 2, `1 <= n <= 2^20` (up to 1M elements)
- Block size must be 256 threads (8 warps of 32 threads)
- Use shared memory for intra-block communication
- The result must match a sequential CPU exclusive scan exactly
- Use `__shfl_up_sync(0xffffffff, val, offset)` for warp-level prefix accumulation
- Do not modify the provided `solve()` function

#include <cuda_runtime.h>

#define BLOCK_SIZE 256

// Single-block exclusive prefix scan using Blelloch algorithm
__global__ void blockScan(const unsigned int* in, unsigned int* out, int n) {
    __shared__ unsigned int tree[2 * BLOCK_SIZE];

    int tid = threadIdx.x;
    int i = blockIdx.x * BLOCK_SIZE + tid;

    // Step 1: Load input into shared memory (upsweep leaves)
    tree[BLOCK_SIZE + tid] = (i < n) ? in[i] : 0;
    __syncthreads();

    // Step 2: Upsweep — build partial sums bottom-up
    // log2(256) = 8 levels for BLOCK_SIZE=256
    for (int d = 0; d < 8; d++) {
        int stride = 1 << (d + 1);  // 2, 4, 8, 16, 32, 64, 128, 256
        if ((tid & (stride - 1)) == (stride / 2)) {
            tree[tid] += tree[tid - stride / 2];
        }
        __syncthreads();
    }

    // Step 3: Set last element to 0 (exclusive scan base case)
    if (tid == 0) tree[2 * BLOCK_SIZE - 1] = 0;
    __syncthreads();

    // Step 4: Downsweep — propagate sums top-down
    for (int d = 7; d >= 0; d--) {  // Reverse order: 7, 6, 5, 4, 3, 2, 1, 0
        int stride = 1 << (d + 1);
        if ((tid & (stride - 1)) == (stride / 2)) {
            unsigned int t = tree[tid - stride / 2];
            tree[tid - stride / 2] = tree[tid];
            tree[tid] += t;
        }
        __syncthreads();
    }

    // Step 5: Store exclusive prefix to output
    out[i] = tree[BLOCK_SIZE + tid];
}

// Extract block sums (last element of each block)
__global__ void extractSums(const unsigned int* blocks,
                            unsigned int* sums, int numBlocks) {
    int bid = blockIdx.x * blockDim.x + threadIdx.x;
    if (bid < numBlocks) {
        sums[bid] = blocks[(bid + 1) * BLOCK_SIZE - 1];
    }
}

void scanPass1(const unsigned int* d_in, unsigned int* d_blocks,
               unsigned int* d_sums, int n) {
    int numBlocks = (n + BLOCK_SIZE - 1) / BLOCK_SIZE;
    blockScan<<<numBlocks, BLOCK_SIZE>>>(d_in, d_blocks, n);

    // Extract block sums (last element of each block)
    extractSums<<<(numBlocks + 255) / 256, 256>>>(d_blocks, d_sums, numBlocks);
}

void scanPass2(unsigned int* d_sums, int numBlocks) {
    // Recursively scan the block sums (single block)
    blockScan<<<1, BLOCK_SIZE>>>(d_sums, d_sums, numBlocks);
}

// Add offset from sums array to each element based on block membership
__global__ void addOffsets(unsigned int* blocks, const unsigned int* offsets,
                           int n, int numBlocks) {
    int tid = blockIdx.x * blockDim.x + threadIdx.x;
    int bid = tid / BLOCK_SIZE;
    int local_idx = tid % BLOCK_SIZE;

    if (tid < n && bid > 0) {
        blocks[tid] += offsets[bid - 1];
    }
}

// Copy result back to output
__global__ void copyOut(const unsigned int* blocks, unsigned int* out, int n) {
    int tid = blockIdx.x * blockDim.x + threadIdx.x;
    if (tid < n) out[tid] = blocks[tid];
}

void scanPass3(const unsigned int* d_blocks, const unsigned int* d_offsets,
               unsigned int* d_out, int n) {
    int numBlocks = (n + BLOCK_SIZE - 1) / BLOCK_SIZE;

    addOffsets<<<(n + 255) / 256, 256>>>(d_blocks, d_offsets, n, numBlocks);
    copyOut<<<(n + 255) / 256, 256>>>(d_blocks, d_out, n);
}

void solve(const unsigned int* h_in, unsigned int* h_out, int n) {
    unsigned int *d_in, *d_out, *d_blocks, *d_sums;

    int numBlocks = (n + BLOCK_SIZE - 1) / BLOCK_SIZE;

    cudaMalloc(&d_in, n * sizeof(unsigned int));
    cudaMalloc(&d_out, n * sizeof(unsigned int));
    cudaMalloc(&d_blocks, numBlocks * BLOCK_SIZE * sizeof(unsigned int));
    cudaMalloc(&d_sums, numBlocks * sizeof(unsigned int));

    cudaMemcpy(d_in, h_in, n * sizeof(unsigned int), cudaMemcpyHostToDevice);

    if (numBlocks == 1) {
        blockScan<<<1, BLOCK_SIZE>>>(d_in, d_out, n);

        scanPass1(d_in, d_blocks, d_sums, n);

        scanPass3(d_blocks, d_sums, d_out, n);
    }

    cudaMemcpy(h_out, d_out, n * sizeof(unsigned int), cudaMemcpyDeviceToHost);
    cudaFree(d_in);
    cudaFree(d_out);
    cudaFree(d_blocks);
    cudaFree(d_sums);
}

## Approach

The Blelloch exclusive scan algorithm consists of two symmetric phases operating on a binary tree structure stored in shared memory.

**Upsweep (reduce phase):**

```cpp
for (int d = 0; d < 8; d++) {  // log2(256) = 8 levels
    int stride = 1 << (d + 1);  // 2, 4, 8, 16, 32, 64, 128, 256
    if ((tid & (stride - 1)) == (stride / 2)) {
        tree[tid] += tree[tid - stride / 2];
    }
    __syncthreads();
}
```

This builds a binary tree from bottom to top. At each level, threads whose index has the pattern `...100...0` (where the rightmost 1 is at position `stride/2`) add their sibling's value to themselves. After 8 levels, `tree[511]` contains the sum of all input elements.

**Downsweep (distribute phase):**

```cpp
if (tid == 0) tree[511] = 0;  // Exclusive scan base case
for (int d = 7; d >= 0; d--) {
    int stride = 1 << (d + 1);
    if ((tid & (stride - 1)) == (stride / 2)) {
        unsigned int t = tree[tid - stride / 2];
        tree[tid - stride / 2] = tree[tid];
        tree[tid] += t;
    }
    __syncthreads();
}
```

This propagates prefix sums top-down. The right child receives the parent's sum; the left child keeps the parent's sum. The key insight is that the right child gets `parent + leftSibling` (which equals the exclusive prefix up to that point), while the left child simply gets `parent`.

**Three-pass algorithm for large arrays:**

1. Each block scans its slice independently, producing partial results and block totals
2. Scan the block totals recursively (single block, at most 256 elements)
3. Add the appropriate offset (sum of all preceding blocks) to each element

**Warp shuffle optimization:**

For the final downsweep step within a warp, replace shared memory access with:
```cpp
for (int offset = 1; offset < 32; offset <<= 1) {
    val += __shfl_up_sync(0xffffffff, val, offset);
}
```

This uses the warp's register file instead of shared memory, eliminating bank conflicts and reducing latency.

## Complexity

- **Work complexity**: O(N) — identical to sequential scan
- **Step complexity**: O(log N) parallel rounds
- **Shared memory**: 2 × BLOCK_SIZE × 4 bytes = 2048 bytes for BLOCK_SIZE=256
- **Three-pass overhead**: Additional global memory reads/writes for large N

## Key Insights

1. **Blelloch vs Hillis-Steele**: Blelloch achieves O(N) work while Hillis-Steele requires O(N log N) work. For N=1M, Blelloch performs ~2M additions while Hillis-Steele performs ~20M — a 10× difference that dominates runtime despite Hillis-Steele's simpler dependency pattern.

2. **Exclusive vs Inclusive**: Exclusive scan outputs 0 at position 0 (no preceding elements). Inclusive scan would output the first element itself. Exclusive scan is essential for computing output indices in stream compaction.

3. **Work efficiency matters on GPUs**: Unlike CPUs where step count (latency) dominates, GPUs execute thousands of threads in parallel. Work-inefficient algorithms waste CUDA Core throughput even if they have few synchronization barriers.

4. **`__shfl_up_sync(0xffffffff, val, offset)`** uses the warp's register file for prefix accumulation. The mask `0xffffffff` enables all 32 lanes; `offset` specifies how many lanes back to read from.

## Common Mistakes

- **Wrong tree size**: The tree array needs 2 × BLOCK_SIZE elements. Leaves are at indices [BLOCK_SIZE, 2×BLOCK_SIZE). Parent of leaf at index i is at index i - BLOCK_SIZE.

- **Missing syncthreads**: Each level of upsweep and downsweep requires synchronization. Without it, threads may read stale data from the previous level.

- **Off-by-one in upsweep**: The loop runs for exactly log2(BLOCK_SIZE) iterations (8 for BLOCK_SIZE=256), not log2(BLOCK_SIZE)-1.

- **Not handling small N**: When n < BLOCK_SIZE, pad with zeros to avoid reading out of bounds. The tree must still be fully built.

- **Wrong offset direction**: `__shfl_up_sync` reads from a lower lane index (up the warp); `__shfl_down_sync` reads from a higher lane index (down the warp). Prefix accumulation uses "up" because each thread needs values from preceding threads.

- **Incorrect three-pass offset**: When adding offsets, use `offsets[bid - 1]` not `offsets[bid]`. Block 0 adds nothing; block 1 adds block 0's sum; block 2 adds blocks 0+1's sum.
