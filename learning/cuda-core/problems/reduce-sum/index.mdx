## Background

**Parallel reduction** is the canonical GPU algorithm for combining N elements into a single value using a binary associative operation (sum, max, min). A naive serial sum takes O(N) time. A well-designed GPU reduction achieves O(N/P) time where P is the number of parallel threads, with only O(log N) synchronization steps.

The fundamental insight is **tree reduction**: pairs of elements are combined in parallel, halving the active count at each step. After log2(N) steps, a single result remains. But naive implementations suffer from three performance killers:

1. **Warp divergence**: When only half the threads are active, the idle threads still consume scheduler slots
2. **Shared memory bank conflicts**: When multiple threads in a warp access adjacent addresses, serialization occurs
3. **Missing thread coarsening**: Each thread loads only one element when it could load many

The evolution of reduction optimization follows five steps, each addressing a specific bottleneck. Understanding this progression is essential for principal-level GPU interview questions.

## What to Implement

Implement `reduceSum` that computes the sum of an array of `n` floats. The optimal approach follows this progression:

**Step 1: Naive stride-1 reduction**
- Load elements into shared memory
- Use `sdata[tid] += sdata[tid + stride]` pattern with increasing stride (1, 2, 4, 8, ...)
- Problem: high-indexed threads are always active, causing divergence

**Step 2: Reverse indexing**
- Lower-indexed threads always active, eliminating divergence
- Pattern: `if (tid < stride) sdata[tid] += sdata[tid + stride]`

**Step 3: Sequential addressing with halving**
- For stride from blockDim/2 down to 1: `if (tid < stride) sdata[tid] += sdata[tid + stride]`
- Coalesced shared memory access

**Step 4: Unroll last warp**
- For threads < 32, use warp-synchronous execution (no __syncthreads inside warp)
- Removes loop overhead for final reduction

**Step 5: Warp shuffle primitives**
- Use `__shfl_down_sync(0xffffffff, val, offset)` instead of shared memory for intra-warp reduction
- Zero shared memory traffic within a warp, 5 instructions (offset = 16, 8, 4, 2, 1)

**Step 6: Thread coarsening**
- Each thread loads multiple elements sequentially before the tree reduction
- Reduces kernel launch overhead and improves arithmetic intensity

Your implementation should use the optimal approach: thread coarsening + warp shuffles + two-pass reduction for large arrays.

## Constraints

- `n` is a power of 2, `1 <= n <= 2^24` (16M elements)
- Use `blockDim.x = 256` (8 warps of 32 threads each)
- Result must match sequential CPU sum within 1e-3 relative error
- Do not modify the provided `solve()` function
- Use `__shfl_down_sync(0xffffffff, val, offset)` for warp-level reduction

__global__ void reduceBlock(const float* in, float* partials, int n) {
    int tid = threadIdx.x;
    int idx = blockIdx.x * BLOCK_SIZE + tid;

    // Thread coarsening: each thread loads one element (could load more)
    float val = (idx < n) ? in[idx] : 0.0f;

    // Warp-level reduction using shuffle (no shared memory)
    for (int offset = WARP_SIZE / 2; offset > 0; offset >>= 1) {
        val += __shfl_down_sync(0xffffffff, val, offset);
    }

    // Write warp partial sum to shared memory (only lane 0 of each warp)
    __shared__ float warp_sums[BLOCK_SIZE / WARP_SIZE];  // 8 floats
    int lane = tid % WARP_SIZE;
    int warp_id = tid / WARP_SIZE;
    if (lane == 0) {
        warp_sums[warp_id] = val;
    }
    __syncthreads();

    // First warp reduces across all warp sums
    if (warp_id == 0) {
        val = (lane < (BLOCK_SIZE / WARP_SIZE)) ? warp_sums[lane] : 0.0f;
        for (int offset = WARP_SIZE / 2; offset > 0; offset >>= 1) {
            val += __shfl_down_sync(0xffffffff, val, offset);
        }
        if (lane == 0) {
            partials[blockIdx.x] = val;
        }
    }
}

__global__ void reducePartials(float* partials, float* out, int count) {
    int tid = threadIdx.x;
    float val = (tid < count) ? partials[tid] : 0.0f;

    // Same pattern as reduceBlock
    for (int offset = WARP_SIZE / 2; offset > 0; offset >>= 1) {
        val += __shfl_down_sync(0xffffffff, val, offset);
    }

    __shared__ float warp_sums[BLOCK_SIZE / WARP_SIZE];
    int lane = tid % WARP_SIZE;
    int warp_id = tid / WARP_SIZE;
    if (lane == 0) {
        warp_sums[warp_id] = val;
    }
    __syncthreads();

    if (warp_id == 0) {
        val = (lane < (BLOCK_SIZE / WARP_SIZE)) ? warp_sums[lane] : 0.0f;
        for (int offset = WARP_SIZE / 2; offset > 0; offset >>= 1) {
            val += __shfl_down_sync(0xffffffff, val, offset);
        }
        if (lane == 0) {
            *out = val;
        }
    }
}

## Approach

Parallel reduction collapses N inputs into one result using a two-pass strategy for large arrays. Each CUDA block independently reduces its slice to a partial sum, then a second kernel pass reduces all partials.

**Within a block, the optimization progression:**

1. **Naive stride-1 reduction** (divergent):
   ```cpp
   for (unsigned int s = 1; s < blockDim.x; s *= 2) {
       if ((tid % (2 * s)) == 0) sdata[tid] += sdata[tid + s];
   }
   ```
   High-indexed threads always active = warp divergence.

2. **Reverse indexing** (divergence-free):
   ```cpp
   for (unsigned int s = blockDim.x / 2; s > 0; s >>= 1) {
       if (tid < s) sdata[tid] += sdata[tid + s];
   }
   ```
   Lower-indexed threads always active = no divergence.

3. **Warp shuffles** (no shared memory intra-warp):
   ```cpp
   for (int offset = WARP_SIZE / 2; offset > 0; offset >>= 1) {
       val += __shfl_down_sync(0xffffffff, val, offset);
   }
   ```
   Uses register file instead of shared memory = zero bank conflicts, minimal latency.

**Two-pass structure for large N:**
- `reduceBlock`: each block reduces `BLOCK_SIZE` elements to one partial sum
- `reducePartials`: single block reduces the partials array

The mask `0xffffffff` in `__shfl_down_sync` means all 32 lanes participate. `__shfl_down_sync(mask, val, offset)` exchanges register values between threads in the same warp without touching shared memory.

**Thread coarsening** (Brent's theorem): each thread loads multiple elements sequentially before the tree reduction. For `n = 1M` and `gridSize = 1024`, each thread loads ~32 elements. This reduces kernel launch overhead and improves arithmetic intensity.

## Complexity

- **Time complexity**: O(N/P) with two passes
  - Pass 1: O(N / BLOCK_SIZE) = O(N / 256)
  - Pass 2: O(gridSize / BLOCK_SIZE) = O(gridSize / 256)
- **Shared memory**: BLOCK_SIZE/WARP_SIZE = 8 floats per block
- **Synchronization**: O(log BLOCK_SIZE) = O(8) __syncthreads() calls per block

## Key Insights

1. **`__shfl_down_sync(0xffffffff, val, offset)`** uses the warp's register file for communication. Offset 16, 8, 4, 2, 1 achieves reduction in exactly 5 instructions, independent of shared memory latency.

2. **Thread coarsening** amortizes kernel launch overhead across more work per thread. If N is large, have each thread load `stride = blockDim * gridDim` elements.

3. **Two-pass reduction** is necessary when N > MAX(gridSize) * BLOCK_SIZE. For N = 16M, gridSize = 65536, and we need a second pass to reduce 65536 partials.

4. **Associativity matters**: The reduction operator must be associative. Floating-point addition is "approximately" associative (within rounding error). Max, min, AND, OR are perfectly associative.

## Common Mistakes

- **Out-of-bounds load**: Always use `(idx < n) ? in[idx] : 0.0f` to handle arrays whose size exceeds the total thread count.

- **Wrong shared memory size**: The `warp_sums` array needs `BLOCK_SIZE / WARP_SIZE` entries (8 for BLOCK_SIZE=256), not `WARP_SIZE` entries (32).

- **Missing __syncthreads() before first-warp reduction**: After writing warp partial sums to shared memory, all 256 threads must complete before the first warp reads from shared memory.

- **Wrong mask in __shfl_down_sync**: Use `0xffffffff` for all 32 lanes, not `0x1f` (which only enables lane 0).

- **Using __shfl without _sync**: The deprecated `__shfl_down(val, offset)` has undefined behavior on Volta+ architectures. Always use `__shfl_down_sync(mask, val, offset)`.
