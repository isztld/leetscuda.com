# FlashAttention Forward (From Scratch)

## Background

The self-attention mechanism is the core building block of Transformer models, enabling the model to dynamically weigh the importance of different tokens in a sequence. For a query matrix Q, key matrix K, and value matrix V, standard attention computes:

```
Attention(Q, K, V) = softmax(Q @ K^T / sqrt(d_k)) @ V
```

where d_k is the head dimension. For sequences of length N, this produces an N×N attention matrix that must be materialized in memory.

### The Memory Bottleneck

Standard attention has O(N²) memory complexity because it stores the full N×N attention matrix in HBM (High Bandwidth Memory). For LLMs with context lengths of 32K, 128K, or even 1M tokens, this becomes prohibitive:

- N = 4096, d_k = 128: attention matrix = 64M elements = 256 MB (FP32)
- N = 32768, d_k = 128: attention matrix = 1B elements = 4 GB (FP32)
- N = 131072, d_k = 128: attention matrix = 17B elements = 68 GB (FP32)

This quadratic scaling severely limits the maximum context length that can be processed on a single GPU.

### Why Standard Attention is IO-Inefficient

The standard attention algorithm performs redundant memory accesses:

1. **Q, K, V are read multiple times**: Q is read once for the QK^T matmul, then K and V are read again for the final V @ attention_output computation.

2. **The N×N attention matrix is written to HBM**: After computing softmax(QK^T), the result is stored in HBM.

3. **The attention matrix is read again**: To compute the final output, the attention matrix is read from HBM and multiplied by V.

Total HBM traffic for standard attention:
- Read Q, K, V: 3 × N × d_k elements
- Write attention matrix: N² elements
- Read attention matrix: N² elements
- Write output: N × d_k elements
- **Total: 4N² + 4Nd_k elements**

For N = 4096, d_k = 128: ~67M elements of HBM traffic.

### The Online Softmax Insight

The key observation behind FlashAttention is that softmax can be computed in a single pass using the **online softmax** algorithm. Instead of:

1. Compute all scores S = QK^T
2. Find max(S) for numerical stability
3. Compute exp(S - max(S))
4. Normalize by sum

We can maintain running statistics (m_i, l_i) where:
- m_i = running maximum of scores seen so far
- l_i = running sum of exp(score - m_i)

When we encounter a new score s:
```
m_new = max(m, s)
l_new = l × exp(m - m_new) + exp(s - m_new)
```

The correction factor exp(m - m_new) rescales previous partial sums when a new maximum is found.

### FlashAttention: Tiled Online Softmax

FlashAttention extends online softmax to the attention mechanism by:

1. **Tiling Q, K, V into blocks** that fit in fast on-chip SRAM (~160KB on H100)
2. **Iterating over K/V blocks**, computing attention scores between Q and each K block
3. **Updating running (m, l) statistics** online as we process each K block
4. **Accumulating partial output O** using the rescaling formula:
   ```
   O_new = O × exp(m_old - m_new) + (exp(S - m_new) @ V)
   ```

5. **Writing only the final O and LSE (log-sum-exp)** to HBM

This reduces HBM traffic to O(N × d_k) instead of O(N²), a massive savings for long sequences.

## What to Implement

You will implement the forward pass of FlashAttention from scratch. Your kernel must:

1. **Partition Q, K, V into tiles** of size B_r × d_k and B_c × d_k that fit in shared memory
2. **For each Q tile**, iterate over all K/V tiles in sequence
3. **Compute attention scores S = Q_i @ K_j^T** in shared memory
4. **Apply causal masking**: S[i,j] = -INF if j > i (for autoregressive attention)
5. **Update running (m, l) statistics** using the online softmax algorithm
6. **Accumulate partial output O** using the rescaling formula
7. **Write final O and LSE (for backward pass)** to global memory

### Algorithm Structure

```
For each Q tile i:
    Load Q_i to shared memory
    Initialize m = -INF, l = 0, O = 0

    For each K/V tile j:
        Load K_j, V_j to shared memory

        # Compute attention scores
        S = Q_i @ K_j^T
        S = S * scale  # 1/sqrt(d_k)
        Apply causal mask: S[i,j] = -INF if j > i

        # Online softmax update
        m_new = max(m, rowmax(S))
        O = O * exp(m - m_new)
        P = exp(S - m_new)
        l = l * exp(m_old - m_new) + rowsum(P)
        O = O + P @ V_j
        m = m_new

    # Write output
    O = O / l
    LSE = m + log(l)
```

### Thread Mapping

Design your kernel with the following parallelism strategy:

- **Grid dimension**: One block per Q tile (gridDim.x = ceil(N / B_r))
- **Block dimension**: 128-256 threads per block, organized as a 2D grid (e.g., 16×8 or 32×4)
- **Each thread** handles a subset of the Q_i tile rows and output columns
- **Within a block**, threads cooperate to load Q, K, V into shared memory and compute partial results

### Suggested Parameters

For this exercise, use the following tile sizes:
- **B_r (Q tile rows)**: 64
- **B_c (K/V tile rows)**: 64
- **Block dimensions**: 16×4 (64 threads) or 16×8 (128 threads)

For hidden dimension d_k = 64 or 128, these parameters ensure:
- Q tile: 64 × 128 × 4 bytes = 32 KB
- K tile: 64 × 128 × 4 bytes = 32 KB
- V tile: 64 × 128 × 4 bytes = 32 KB
- S matrix: 64 × 64 × 4 bytes = 16 KB
- **Total: ~132 KB** (within 160 KB SRAM on H100)

### Causal Masking

FlashAttention is commonly used with causal (autoregressive) masking in LLMs. For causal attention:
- Query at position i can only attend to keys at positions j ≤ i
- Set S[i,j] = -INF for all j > i before applying softmax

The causal mask is applied **per Q tile** relative to the K tile being processed:
- If Q tile covers rows [i*B_r, (i+1)*B_r) and K tile covers rows [j*B_c, (j+1)*B_c)
- Then S[local_i, local_j] = -INF if (j*B_c + local_j) > (i*B_r + local_i)

### Output: LSE for Backward Pass

Your kernel must also compute and store the **log-sum-exp (LSE)** values:
```
LSE[i] = m[i] + log(l[i])
```

These LSE values are required for the backward pass (which you do not need to implement). The backward pass uses LSE to efficiently recompute the attention weights without storing the full N×N matrix.

## Constraints

### Rules

1. **Implement exact attention**: Your output must match PyTorch's `torch.nn.functional.scaled_dot_product_attention` with `causal=True` to within 1e-4 absolute error (FP32).

2. **Do not materialize the N×N attention matrix** in global memory. The only N²-sized data structure you may use is a **shared memory tile** (S matrix) of size B_r × B_c.

3. **Use online softmax**: You must update (m, l) statistics incrementally as you process each K tile, not in a separate pass.

4. **Support any sequence length N** that is a multiple of B_r (use padding for the last tile if needed).

5. **Use float32** for all computations (no FP16/BF16).

6. **Compute softmax_scale = 1.0 / sqrt(d_k)** inside the kernel.

### Input Data Layout

Your `solve()` function receives the following inputs:
- `Q`: Query matrix of shape (batch_size, seqlen, nheads, head_dim)
- `K`: Key matrix of shape (batch_size, seqlen, nheads, head_dim)
- `V`: Value matrix of shape (batch_size, seqlen, nheads, head_dim)
- `out`: Output buffer of shape (batch_size, seqlen, nheads, head_dim)
- `lse`: LSE buffer of shape (batch_size, nheads, seqlen)

All matrices are in **row-major layout**. For simplicity, you may assume:
- `batch_size = 1`
- `nheads = 1` (single-head attention)
- `head_dim` is 64 or 128

### Performance Target

Your implementation should achieve at least **10% of the theoretical peak HBM bandwidth** for the attention operation. For an H100 with 3 TB/s HBM bandwidth:
- Theoretical minimum time for N=4096, d=128: 4N² + 4Nd = 67M elements × 4 bytes = 268 MB
- At 10% bandwidth (300 GB/s): ~0.9 ms minimum
- Target: **< 10 ms** for the forward pass

Note: A naive implementation that materializes the N×N matrix will be significantly slower due to the O(N²) HBM traffic.

#include <cuda_runtime.h>
#include <cstdio>
#include <cmath>
#include <algorithm>
#include <vector>

// Tile sizes - must fit in shared memory (~160 KB on H100)
constexpr int B_r = 64;  // Q tile rows
constexpr int B_c = 64;  // K/V tile rows
constexpr int BLOCK_DIM_X = 16;
constexpr int BLOCK_DIM_Y = 8;  // 128 threads per block

// Shared memory is dynamically allocated in the kernel launch
__global__ void flash_attention_kernel(
    const float* __restrict__ Q,
    const float* __restrict__ K,
    const float* __restrict__ V,
    float* __restrict__ out,
    float* __restrict__ lse,
    int batch_size,
    int seqlen,
    int nheads,
    int head_dim,
    float softmax_scale
) {
    // Thread and block indices
    const int tid_x = threadIdx.x;
    const int tid_y = threadIdx.y;
    const int bdx = blockDim.x;
    const int bdy = blockDim.y;
    const int block_idx = blockIdx.x;

    // Batch and head indices (for simplicity, this example assumes batch=1, nheads=1)
    const int b = blockIdx.y;
    const int h = blockIdx.z;

    // Shared memory declaration (sizes computed at compile time)
    extern __shared__ float smem[];
    float* Q_tile = smem;
    float* K_tile = &Q_tile[B_r * head_dim];
    float* V_tile = &K_tile[B_c * head_dim];
    float* S_tile = &V_tile[B_c * head_dim];

    // Per-thread registers for running statistics
    // Each thread handles multiple rows of the Q tile
    const int rows_per_thread = (B_r + bdy - 1) / bdy;
    float m[rows_per_thread];  // running max
    float l[rows_per_thread];  // running sum of exp

    // Initialize m = -INF, l = 0
    for (int r = 0; r < rows_per_thread; r++) {
        int row_idx = tid_y * rows_per_thread + r;
        if (row_idx < B_r) {
            m[r] = -1e30f;
            l[r] = 0.0f;
        }
    }

    // Output accumulator O (stored in shared memory for this example)
    float* O_tile = &S_tile[B_r * B_c];
    __shared__ float O_accum[B_r * head_dim];

    // Initialize O = 0
    for (int r = 0; r < rows_per_thread; r++) {
        int row_idx = tid_y * rows_per_thread + r;
        if (row_idx < B_r) {
            for (int c = tid_x; c < head_dim; c += bdx) {
                O_accum[row_idx * head_dim + c] = 0.0f;
            }
        }
    }
    __syncthreads();

    // Compute Q tile row range
    int q_start = block_idx * B_r;
    int q_end = min(q_start + B_r, seqlen);
    int actual_B_r = q_end - q_start;

    // Base pointer offset for this batch and head
    int batch_head_offset = (b * nheads * seqlen * head_dim) + (h * seqlen * head_dim);
    int lse_offset = (b * nheads * seqlen) + h * seqlen;

    // Load Q tile into shared memory
    for (int r = tid_y; r < actual_B_r; r += bdy) {
        for (int c = tid_x; c < head_dim; c += bdx) {
            Q_tile[r * head_dim + c] = Q[batch_head_offset + (q_start + r) * head_dim + c];
        }
    }
    __syncthreads();

    // Accumulate output over all K/V tiles
    int num_k_tiles = (seqlen + B_c - 1) / B_c;

    for (int j = 0; j < num_k_tiles; j++) {
        int k_start = j * B_c;
        int k_end = min(k_start + B_c, seqlen);
        int actual_B_c = k_end - k_start;

        // Load K tile and V tile
        for (int r = tid_y; r < actual_B_c; r += bdy) {
            for (int c = tid_x; c < head_dim; c += bdx) {
                int global_idx = batch_head_offset + (k_start + r) * head_dim + c;
                K_tile[r * head_dim + c] = K[global_idx];
                V_tile[r * head_dim + c] = V[global_idx];
            }
        }
        __syncthreads();

        // Compute S = Q @ K^T
        // Each thread computes a subset of S[row, col]
        for (int row = tid_x; row < actual_B_r; row += bdx) {
            for (int col = 0; col < actual_B_c; col += bdy) {
                float sum = 0.0f;
                for (int d = 0; d < head_dim; d++) {
                    sum += Q_tile[row * head_dim + d] * K_tile[col * head_dim + d];
                }
                sum *= softmax_scale;

                // Apply causal mask
                int global_q = q_start + row;
                int global_k = k_start + col;
                if (global_k > global_q) {
                    sum = -1e30f;
                }

                S_tile[row * B_c + col] = sum;
            }
        }
        __syncthreads();

        // Update running statistics (online softmax)
        // Each thread updates its assigned rows
        for (int r = 0; r < rows_per_thread; r++) {
            int row_idx = tid_y * rows_per_thread + r;
            if (row_idx >= actual_B_r) continue;

            float row_max = m[r];

            // Find max across columns for this row
            for (int col = 0; col < actual_B_c; col++) {
                row_max = fmaxf(row_max, S_tile[row_idx * B_c + col]);
            }

            float m_old = m[r];
            m[r] = row_max;

            // Rescale O and accumulate
            float scale = expf(m_old - m[r]);

            float row_sum = 0.0f;
            for (int col = 0; col < actual_B_c; col++) {
                float P = expf(S_tile[row_idx * B_c + col] - m[r]);
                row_sum += P;

                // O = O * scale + P * V
                for (int d = tid_x; d < head_dim; d += bdx) {
                    O_accum[row_idx * head_dim + d] =
                        O_accum[row_idx * head_dim + d] * scale + P * V_tile[col * head_dim + d];
                }
            }

            l[r] = l[r] * scale + row_sum;
        }
        __syncthreads();
    }

    // Write output: O = O / l
    // Use the same blocked row assignment as the accumulation loop above.
    for (int r = 0; r < rows_per_thread; r++) {
        int row_idx = tid_y * rows_per_thread + r;
        if (row_idx >= actual_B_r) continue;

        for (int c = tid_x; c < head_dim; c += bdx) {
            int global_idx = batch_head_offset + (q_start + row_idx) * head_dim + c;
            out[global_idx] = O_accum[row_idx * head_dim + c] / l[r];
        }

        // Write LSE = m + log(l)
        if (tid_x == 0) {
            int global_q = q_start + row_idx;
            lse[lse_offset + global_q] = m[r] + logf(l[r]);
        }
    }
}

void solve(const float* hQ, const float* hK, const float* hV,
           float* hOut, float* hLSE,
           int batch_size, int seqlen, int nheads, int head_dim) {

    // Compute softmax scale
    float softmax_scale = 1.0f / sqrtf((float)head_dim);

    // Allocate device memory
    float *dQ, *dK, *dV, *dOut, *dLSE;
    size_t qkv_size = batch_size * seqlen * nheads * head_dim * sizeof(float);
    size_t lse_size = batch_size * nheads * seqlen * sizeof(float);

    cudaMalloc(&dQ, qkv_size);
    cudaMalloc(&dK, qkv_size);
    cudaMalloc(&dV, qkv_size);
    cudaMalloc(&dOut, qkv_size);
    cudaMalloc(&dLSE, lse_size);

    // Copy inputs to device
    cudaMemcpy(dQ, hQ, qkv_size, cudaMemcpyHostToDevice);
    cudaMemcpy(dK, hK, qkv_size, cudaMemcpyHostToDevice);
    cudaMemcpy(dV, hV, qkv_size, cudaMemcpyHostToDevice);

    // Compute grid and block dimensions
    int num_q_tiles = (seqlen + B_r - 1) / B_r;
    dim3 block_dim(BLOCK_DIM_X, BLOCK_DIM_Y);  // 128 threads
    dim3 grid_dim(num_q_tiles, batch_size, nheads);

    // Compute shared memory size
    size_t shared_mem_size =
        (B_r * head_dim +      // Q tile
         B_c * head_dim +      // K tile
         B_c * head_dim +      // V tile
         B_r * B_c +           // S matrix
         B_r * head_dim) *     // O accumulator
        sizeof(float);

    // Launch kernel
    flash_attention_kernel<<<grid_dim, block_dim, shared_mem_size>>>(
        dQ, dK, dV, dOut, dLSE,
        batch_size, seqlen, nheads, head_dim, softmax_scale
    );

    // Copy output back to host
    cudaMemcpy(hOut, dOut, qkv_size, cudaMemcpyDeviceToHost);
    cudaMemcpy(hLSE, dLSE, lse_size, cudaMemcpyDeviceToHost);

    // Cleanup
    cudaFree(dQ);
    cudaFree(dK);
    cudaFree(dV);
    cudaFree(dOut);
    cudaFree(dLSE);
}

// CPU reference implementation (naive attention)
// Uses a separate score buffer to avoid aliasing with the output buffer.
void reference_attention(const float* Q, const float* K, const float* V,
                         float* out, int batch_size, int seqlen, int nheads, int head_dim) {
    float scale = 1.0f / sqrtf((float)head_dim);
    // Allocate separate score buffer to avoid overwriting out mid-computation
    std::vector<float> scores(seqlen);

    for (int b = 0; b < batch_size; b++) {
        for (int h = 0; h < nheads; h++) {
            for (int i = 0; i < seqlen; i++) {
                // Compute attention scores into separate buffer
                for (int j = 0; j < seqlen; j++) {
                    float score = 0.0f;
                    for (int d = 0; d < head_dim; d++) {
                        int q_idx = ((b * seqlen + i) * nheads + h) * head_dim + d;
                        int k_idx = ((b * seqlen + j) * nheads + h) * head_dim + d;
                        score += Q[q_idx] * K[k_idx];
                    }
                    score *= scale;
                    // Causal mask
                    if (j > i) score = -1e30f;
                    scores[j] = score;
                }

                // Softmax over scores
                float max_score = -1e30f;
                for (int j = 0; j < seqlen; j++) max_score = fmaxf(max_score, scores[j]);

                float sum_exp = 0.0f;
                for (int j = 0; j < seqlen; j++) {
                    scores[j] = expf(scores[j] - max_score);
                    sum_exp += scores[j];
                }

                // Normalize and compute output into out buffer
                for (int d = 0; d < head_dim; d++) {
                    int out_idx = ((b * seqlen + i) * nheads + h) * head_dim + d;
                    out[out_idx] = 0.0f;
                    for (int j = 0; j < seqlen; j++) {
                        int v_idx = ((b * seqlen + j) * nheads + h) * head_dim + d;
                        out[out_idx] += scores[j] / sum_exp * V[v_idx];
                    }
                }
            }
        }
    }
}

        float max_error = 0.0f;
        for (int i = 0; i < qkv_size; i++) {
            float error = fabsf(hOut_gpu[i] - hOut_ref[i]);
            max_error = fmaxf(max_error, error);
        }

        bool pass = max_error < 1e-4f;
        printf("%d\n", pass ? 1 : 0);
        if (!pass) {
            fprintf(stderr, "Max error: %e (threshold: 1e-4)\n", max_error);
        }

    } else {
        // Explicit input mode
        int batch_size = atoi(first);
        int seqlen, head_dim;
        scanf("%d %d", &seqlen, &head_dim);

        int nheads = 1;
        int qkv_size = batch_size * seqlen * nheads * head_dim;

        bool fill_mode;
        float fill_val;

        std::vector<float> hQ, hK, hV;
        readMatrix(hQ, batch_size * seqlen * nheads, head_dim, fill_mode, fill_val);
        readMatrix(hK, batch_size * seqlen * nheads, head_dim, fill_mode, fill_val);
        readMatrix(hV, batch_size * seqlen * nheads, head_dim, fill_mode, fill_val);

        std::vector<float> hOut(qkv_size), hLSE(batch_size * nheads * seqlen);
        solve(hQ.data(), hK.data(), hV.data(), hOut.data(), hLSE.data(),
              batch_size, seqlen, nheads, head_dim);

        // Print output matrix
        for (int i = 0; i < qkv_size; i++) {
            if (i > 0) printf(" ");
            printf("%.4f", hOut[i]);
        }
        printf("\n");
    }

    return 0;
}

## Approach

FlashAttention is a breakthrough algorithm that transforms attention from O(N²) memory to O(N) memory while maintaining exact numerical equivalence with standard attention. The key insight is **IO-awareness**: the algorithm is designed to minimize expensive HBM (High Bandwidth Memory) accesses by maximizing reuse of data in fast on-chip SRAM.

### Core Algorithm

The FlashAttention forward pass follows this structure:

1. **Tile Q into B_r-sized blocks**. Each Q tile is processed independently and written to output.

2. **For each Q tile, iterate over all K/V tiles**:
   - Load K_j, V_j tiles into SRAM
   - Compute S = Q_i @ K_j^T in SRAM
   - Apply causal mask: S[i,j] = -INF if j > i
   - Update running (m, l) statistics using online softmax
   - Accumulate O = O × exp(m_old - m_new) + exp(S - m_new) @ V_j

3. **After processing all K tiles**, write O / l to output and LSE = m + log(l) for backward pass.

### Why It's Correct

The online softmax algorithm maintains the invariant:
```
l_i = Σ_j exp(S_j - m_i)
O_i = Σ_j exp(S_j - m_i) @ V_j
```

When a new score S_new is encountered with max m_new = max(m_i, S_new):
```
l_new = l_i × exp(m_i - m_new) + exp(S_new - m_new)
      = Σ_j exp(S_j - m_new) + exp(S_new - m_new)
      = Σ_{j, new} exp(S - m_new)

O_new = O_i × exp(m_i - m_new) + exp(S_new - m_new) @ V_new
      = Σ_j exp(S_j - m_new) @ V_j + exp(S_new - m_new) @ V_new
      = Σ_{j, new} exp(S - m_new) @ V
```

This is exactly the online softmax update, which is mathematically equivalent to computing softmax over all scores at once.

### Complexity Analysis

**Memory (HBM traffic):**
- Standard attention: 4N² + 4Nd elements
- FlashAttention: 4Nd + 4Nd = 8Nd elements (read Q,K,V once, write O once)
- **Speedup factor**: O(N/d) for long sequences

**Computation:**
- Standard attention: 2N²d FLOPs (QK^T) + 2N²d FLOPs (attention @ V) = 4N²d FLOPs
- FlashAttention: Same FLOPs, but reorganized into tiles
- **FLOPs are identical**; only memory access pattern changes

**Time complexity:**
- Standard: O(N²) dominated by HBM traffic
- FlashAttention: O(N) for long sequences

### Common Mistakes

1. **Incorrect causal mask**: The causal mask must compare **global** positions, not tile-local positions. If Q tile i covers [i*B_r, (i+1)*B_r) and K tile j covers [j*B_c, (j+1)*B_c), then S[local_i, local_j] is masked if j*B_c + local_j > i*B_r + local_i.

2. **Forgetting to rescale O**: When updating m, you must multiply the existing O by exp(m_old - m_new). Without this rescaling, the output will be incorrect.

3. **Numerical overflow in exp()**: Always subtract the running max before calling exp(). Even within a tile, scores can exceed 700 (overflow for FP32 exp).

4. **Incorrect LSE computation**: LSE = m + log(l), not just log(l). The m term is crucial for numerical stability in the backward pass.

5. **Shared memory overflow**: With B_r = B_c = 64 and head_dim = 128, you need ~132 KB shared memory. Ensure your kernel launch requests enough shared memory.

### Key Insights

1. **Exact attention, not approximation**: FlashAttention computes the exact same output as standard attention. It doesn't use low-rank approximations or sparse patterns (unless explicitly added).

2. **Backward pass is also O(N)**: The backward pass recomputes attention weights on-the-fly using stored LSE values. This is the key to memory-efficient training.

3. **Tile size matters**: B_r and B_c must be large enough for good utilization but small enough to fit in SRAM along with Q, K, V, and S tiles.

4. **Works with any mask**: The algorithm generalizes to arbitrary attention masks (sliding window, block sparse, etc.) by changing the masking condition.

### Further Optimization

For production use, consider:

- **Vectorized loads/stores**: Use float4 reads for 4× bandwidth
- **Double buffering**: Overlap K/V loads with computation
- **Warp-level primitives**: Use __shfl_sync for intra-warp reductions
- **Tensor Cores**: Use WMMA instructions for Q@K^T matmul (FA3 does this)
- **TMA (Tensor Memory Accelerator)**: Hopper's async copy for zero-overhead loads (FA3)
