## Background

The **Roofline model** is a visual performance model that characterises whether a computation is *compute-bound* or *memory-bandwidth-bound*. The key metric is **arithmetic intensity** (AI): the ratio of floating-point operations to bytes of memory traffic.

```
Arithmetic Intensity (AI) = FLOPs / bytes_moved
```

A GPU has two fundamental limits:
- **Peak compute** (FLOP/s): e.g., 312 TFLOP/s (FP16) on A100
- **Peak memory bandwidth** (bytes/s): e.g., 2 TB/s on A100

The roofline for a kernel is: `attainable_perf = min(peak_compute, AI × peak_bandwidth)`

If a kernel's AI lies left of the **ridge point** (`peak_compute / peak_bandwidth`), it is memory-bandwidth-bound. Operations like vector-add (AI ≈ 0.17 FLOP/byte) are always bandwidth-bound; matrix multiply (AI ∝ tile size) becomes compute-bound once tiles are large enough.

## What to Implement

Implement the following analysis functions in C++:

```cpp
// Compute arithmetic intensity given FLOPs and bytes moved
double arithmetic_intensity(double flops, double bytes_moved);

// Attainable performance given AI, peak FLOP/s, and peak bandwidth (bytes/s)
double attainable_perf(double ai, double peak_flops, double peak_bandwidth);

// True if the kernel is compute-bound (AI >= ridge point)
bool is_compute_bound(double ai, double peak_flops, double peak_bandwidth);

// Ridge point: the AI at which compute and bandwidth limits intersect
double ridge_point(double peak_flops, double peak_bandwidth);
```

Also implement `analyse_kernel` that returns a summary:

```cpp
struct RooflineResult {
    double ai;                  // arithmetic intensity (FLOP/byte)
    double attainable_gflops;   // attainable performance in GFLOP/s
    bool compute_bound;
    double ridge;               // ridge point (FLOP/byte)
};

RooflineResult analyse_kernel(
    double flops, double bytes_moved,
    double peak_gflops, double peak_gbps);
```

## Constraints

- All inputs are positive doubles
- `peak_gflops` and `peak_gbps` are in GFLOP/s and GB/s respectively (not raw FLOP/s or bytes/s)
- Return `attainable_gflops` capped at `peak_gflops`

## Example

NVIDIA A100 SXM4 (FP32): peak_gflops = 19,500, peak_gbps = 2,000

Matrix multiply with n=4096: flops = 2×4096³ ≈ 1.37×10¹¹, bytes = 3×4096²×4 ≈ 201 MB → AI ≈ 683 FLOP/byte

```
ridge_point(19500, 2000) = 9.75 FLOP/byte
AI 683 >> 9.75 → compute-bound ✓
attainable_perf = min(19500, 683 × 2000) = 19500 GFLOP/s
```

double arithmetic_intensity(double flops, double bytes_moved) {
    return flops / bytes_moved;
}

double ridge_point(double peak_gflops, double peak_gbps) {
    return peak_gflops / peak_gbps;
}

bool is_compute_bound(double ai, double peak_gflops, double peak_gbps) {
    return ai >= ridge_point(peak_gflops, peak_gbps);
}

double attainable_perf(double ai, double peak_gflops, double peak_gbps) {
    return std::min(peak_gflops, ai * peak_gbps);
}

RooflineResult analyse_kernel(
    double flops, double bytes_moved,
    double peak_gflops, double peak_gbps)
{
    double ai = arithmetic_intensity(flops, bytes_moved);
    return {
        ai,
        attainable_perf(ai, peak_gflops, peak_gbps),
        is_compute_bound(ai, peak_gflops, peak_gbps),
        ridge_point(peak_gflops, peak_gbps),
    };
}

            double ai, pg, pb;
            scanf("%lf %lf %lf", &ai, &pg, &pb);
            printf("%d\n", is_compute_bound(ai, pg, pb) ? 1 : 0);
        } else if (strcmp(cmd, "PERF") == 0) {
            double ai, pg, pb;

            printf("%.4f\n", attainable_perf(ai, pg, pb));
        } else if (strcmp(cmd, "ANALYSE") == 0) {
            double flops, bytes, pg, pb;

                   r.ai, r.attainable_gflops,
                   r.compute_bound ? 1 : 0,
                   r.ridge);
        }
    }

}

---editorial---
## Approach

Each function is a direct translation of the roofline model equations:

```cpp
double arithmetic_intensity(double flops, double bytes_moved) {
    return flops / bytes_moved;   // FLOP/byte
}

double ridge_point(double peak_gflops, double peak_gbps) {
    return peak_gflops / peak_gbps;   // FLOP/byte at which lines intersect
}

bool is_compute_bound(double ai, double peak_gflops, double peak_gbps) {
    return ai >= ridge_point(peak_gflops, peak_gbps);
}

double attainable_perf(double ai, double peak_gflops, double peak_gbps) {
    return std::min(peak_gflops, ai * peak_gbps);
}
```

`ai * peak_gbps` is the memory-bandwidth-limited throughput (how many GFLOP/s you can sustain given the available bytes/s and the computation density). It's capped at `peak_gflops` — you can't exceed peak compute regardless of how much data you feed.

## Complexity

All functions are O(1).

## Key Insight

The ridge point `peak_gflops / peak_gbps` has units of FLOP/byte. It's the arithmetic intensity at which the two constraints are exactly equal: a kernel at this AI saturates both compute and memory simultaneously. Kernels left of the ridge are memory-bound; right of the ridge are compute-bound.

For the A100 SXM4 (FP32): ridge = 19,500 / 2,000 = **9.75 FLOP/byte**. A simple vector addition (AI ≈ 0.17) is ~57× below the ridge — clearly memory-bound.

## Common Mistakes

- **Units**: The function takes `peak_gflops` (GFLOP/s) and `peak_gbps` (GB/s), so the ratio is already in FLOP/byte. Don't convert to raw FLOP/s or bytes/s.
- **Not capping at peak_gflops**: A highly compute-bound kernel's theoretical performance is `ai * peak_gbps`, but it can never exceed the hardware's peak throughput.
