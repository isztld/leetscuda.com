## Background

Data must travel across the **PCIe bus** before a GPU can process it. Host-to-Device (H2D) and Device-to-Host (D2H) transfers via `cudaMemcpy` are the most common bottleneck in data-loading pipelines. PCIe 4.0 ×16 offers ~32 GB/s of peak unidirectional bandwidth; PCIe 5.0 ×16 doubles that to ~64 GB/s. In practice, pinned (page-locked) host memory is required to achieve peak transfer rates — pageable memory incurs an internal copy through a staging buffer.

**Pinned memory** (allocated with `cudaHostAlloc` or `cudaMallocHost`) is locked in physical RAM and can be transferred directly by the DMA engine, bypassing the staging step. Large contiguous transfers (≥1 MB) approach the theoretical bus bandwidth; small transfers are dominated by per-transfer overhead (~5–10 µs).

## What to Implement

Implement `measureBandwidth` that measures H2D and D2H bandwidth in GB/s:

```cpp
struct BandwidthResult {
    double h2d_gbps;  // host-to-device bandwidth
    double d2h_gbps;  // device-to-host bandwidth
};

BandwidthResult measureBandwidth(size_t bytes, int iterations);
```

For each direction:
1. Allocate pinned host memory with `cudaMallocHost` and device memory with `cudaMalloc`
2. Warm up with 3 transfers (to prime the PCIe link)
3. Time `iterations` transfers using `cudaEvent` timing (`cudaEventRecord` / `cudaEventSynchronize` / `cudaEventElapsedTime`)
4. Compute bandwidth: `bytes * iterations / elapsed_seconds / 1e9`
5. Free all allocations

## Constraints

- Use `cudaEventRecord` for timing (not `clock()` or `gettimeofday`)
- Use `cudaMallocHost` for pinned host memory (not `malloc`)
- `bytes` is a multiple of 512, `1 ≤ iterations ≤ 1000`
- Return double-precision GB/s (bytes per second / 1e9)

## Example

On a PCIe 4.0 system with a modern GPU:

```
measureBandwidth(64 * 1024 * 1024, 100)
→ { h2d_gbps: 26.4, d2h_gbps: 25.8 }
```

---solution---
BandwidthResult measureBandwidth(size_t bytes, int iterations) {
    BandwidthResult result = {0.0, 0.0};
    char *h_buf, *d_buf;
    cudaMallocHost(&h_buf, bytes);
    cudaMalloc(&d_buf, bytes);
    cudaEvent_t start, stop;
    cudaEventCreate(&start);
    cudaEventCreate(&stop);

    // Warm up H2D
    for (int i = 0; i < 3; ++i)
        cudaMemcpy(d_buf, h_buf, bytes, cudaMemcpyHostToDevice);

    // Time H2D
    cudaEventRecord(start);
    for (int i = 0; i < iterations; ++i)
        cudaMemcpy(d_buf, h_buf, bytes, cudaMemcpyHostToDevice);
    cudaEventRecord(stop);
    cudaEventSynchronize(stop);
    float ms = 0;
    cudaEventElapsedTime(&ms, start, stop);
    result.h2d_gbps = (double)bytes * iterations / (ms / 1000.0) / 1e9;

    // Warm up D2H
    for (int i = 0; i < 3; ++i)
        cudaMemcpy(h_buf, d_buf, bytes, cudaMemcpyDeviceToHost);

    // Time D2H
    cudaEventRecord(start);
    for (int i = 0; i < iterations; ++i)
        cudaMemcpy(h_buf, d_buf, bytes, cudaMemcpyDeviceToHost);
    cudaEventRecord(stop);
    cudaEventSynchronize(stop);
    cudaEventElapsedTime(&ms, start, stop);
    result.d2h_gbps = (double)bytes * iterations / (ms / 1000.0) / 1e9;

    cudaEventDestroy(start);
    cudaEventDestroy(stop);
    cudaFreeHost(h_buf);
    cudaFree(d_buf);
    return result;
}

void solve(size_t bytes_mb, int iterations) {
    size_t bytes = bytes_mb * 1024 * 1024;
    bytes = (bytes / 512) * 512;
    if (bytes == 0) bytes = 512;
    BandwidthResult r = measureBandwidth(bytes, iterations);
    int ok = (r.h2d_gbps > 0.5 && r.h2d_gbps < 1000.0 &&
              r.d2h_gbps > 0.5 && r.d2h_gbps < 1000.0) ? 1 : 0;
    printf("%d\n", ok);
}

## Approach

Measuring PCIe bandwidth correctly requires three things: pinned host memory, CUDA event timing, and warm-up transfers.

**Why pinned memory?** `malloc`'d memory is pageable — the GPU DMA engine cannot access it directly. CUDA internally copies pageable memory to a pinned staging buffer before the transfer, adding latency and roughly halving bandwidth. `cudaMallocHost` allocates physically pinned pages the DMA engine can access directly.

**Why CUDA events?** CPU-side clocks (`clock()`, `chrono`) measure wall time including scheduling jitter. CUDA events are recorded on the GPU command stream and their elapsed time reflects only the GPU's view of the transfer duration — accurate to ~0.5 µs.

**Bandwidth formula:**
```
bandwidth = (bytes × iterations) / (elapsed_ms / 1000) / 1e9  [GB/s]
```

**Warm-up:** The PCIe link negotiates its state on first use. The first few transfers are slower while the link ramps up. Three warm-up transfers are sufficient to prime the link.

## Complexity

- **Time**: O(iterations) per direction
- **Accuracy**: grows with iterations (averaging reduces clock noise)

## Key Insight

H2D and D2H bandwidth are independently measured because they use separate DMA engines and can differ by 5–15% depending on the system. PCIe is full-duplex but most systems cannot achieve full bidirectional bandwidth simultaneously due to credit-based flow control.

## Common Mistakes

- **Using pageable malloc instead of `cudaMallocHost`**: Bandwidth will be ~50% of the correct value.
- **Not calling `cudaEventSynchronize(stop)` before `cudaEventElapsedTime`**: Without sync, elapsed time is undefined (the stop event may not have been recorded yet).
- **Forgetting to destroy events and free memory**: CUDA events consume device resources; leaking them across repeated calls accumulates overhead.
